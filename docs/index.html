dev-3.0

Terminal-centric project manager for AI coding agents

Manage dozens of AI agents in parallel. Each task gets its own isolated git worktree, a real terminal with tmux, and your choice of coding agent — all organized on a beautiful Kanban board.

Why dev-3.0?

From chaos to clarity

The Problem

Running multiple AI agents means juggling dozens of terminal windows, losing track of which agent works on what, and dealing with git conflicts between parallel changes.

The Solution

dev-3.0 gives every task its own isolated git worktree and terminal. A Kanban board shows what each agent is doing. No conflicts, no chaos — just organized parallel development.

Workflow

How it works

Four steps from idea to merged code.

01

Create a Task

Add a task to the Kanban board. Give it a name, pick a base branch.

02

Pick Your Agent

Choose from Claude, Codex, Gemini, or any terminal-based agent. Configure model, permissions, and budget.

03

Work in Isolation

Each task spawns an isolated git worktree with its own terminal running tmux. Full power, zero conflicts.

04

Review & Complete

Review the work, merge or discard. Worktree and terminal are cleaned up automatically.

Features

Everything you need to orchestrate AI agents

Kanban Board

7-column workflow from To Do to Completed. Drag-and-drop cards, terminal preview on hover, real-time agent status.

Git Worktrees

Every active task gets its own git worktree — a fully isolated branch and working directory. No stashing, no conflicts.

Multi-Agent Support

Built-in support for Claude Code, OpenAI Codex, and Google Gemini. Configure models, permissions, effort levels, and budgets per agent.

Real Terminal

Powered by ghostty-web and tmux. Full terminal emulation, not a toy. Every feature your agent supports works out of the box.

Task Variants

Run the same task with different agents or configurations in parallel. Compare approaches side by side.

Project Config

Setup scripts run automatically in new worktrees. Configure base branches, cleanup scripts, and dev servers per project.
