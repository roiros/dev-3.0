Fix Cmd+H not hiding the app on macOS. WKWebView (used by Electrobun) intercepts keyboard events before they reach the native menu system, so the `{ role: "hide" }` menu item's Cmd+H accelerator was silently swallowed. Added a JavaScript capture-phase handler that intercepts Cmd+H and calls the new `hideApp` RPC handler, which invokes `[NSApp hide:nil]` directly via Objective-C runtime FFI (no permissions required). Closes #162.
