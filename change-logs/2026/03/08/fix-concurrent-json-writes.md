Add file-level locking (mkdir-based atomic lock) to all JSON read-modify-write operations in the data layer. This prevents concurrent writes from losing updates — the root cause of issue #142 (worktrees randomly disappearing). All mutator functions in data.ts (addTask, updateTask, deleteTask, addProject, removeProject, updateProject, reorderTasksInColumn, setLastPickedFolder, saveTasks, saveProjects) now serialize through withFileLock(). Includes 21 new tests: 11 for file-lock mechanics (mutual exclusion, stale recovery, timeout) and 10 for concurrent data operations (parallel addTask, mixed add/update/delete).
