Fix default branch detection when adding a project. Previously, if `refs/remotes/origin/HEAD` was not set (common when a remote is added manually rather than via clone), the fallback only checked local branches and often picked the wrong one. Now the app auto-sets `origin/HEAD` via `git remote set-head --auto`, then falls back to checking remote tracking branches. Also fixed the diff view to compare against `origin/<baseBranch>` consistently with the branch status panel, and to respect the user-selected compare ref.
