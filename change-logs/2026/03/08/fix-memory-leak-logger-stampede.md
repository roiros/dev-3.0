Fix critical memory leak caused by three compounding issues: (1) Logger `appendToFile()` read the entire log file and rewrote it for every single line — O(n²) memory allocation that reached ~44MB/sec at peak logging rates. Replaced with `appendFileSync` and cached `mkdirSync`. (2) `getBranchStatus` RPC had no dedup — on app wake/reconnect, dozens of duplicate calls for the same task fired simultaneously, each spawning 16+ git subprocesses and reading full `git log -p` output into JS strings. Added in-flight request deduplication. (3) `isContentMergedInto` Strategy 2 stored full `git log -p` and `git diff` output as JS strings (potentially multi-MB each). Replaced with shell pipes (`git log -p | git patch-id`) so patch data never enters JS memory.
