Fix column order persistence: cross-column moved tasks now get proper `columnOrder` assigned immediately based on `taskDropPosition` setting (top/bottom), so their position survives app reload. Previously, cross-column moves relied on ephemeral `moveOrderMap` (lost on reload), causing tasks to jump positions. Added `dropPosition` option to `updateTask` in the data layer, wired through both RPC handler and CLI socket server. Includes 52 new tests covering reorder persistence, reload scenarios, and top/bottom mode.
