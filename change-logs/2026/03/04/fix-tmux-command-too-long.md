Fix "command too long" error on terminal restart. tmux 3.x limits the shell-command for `new-session` to ~16 KB. The old `buildEchoAndRun` doubled the command (once for echo, once for execution), easily exceeding that limit for tasks with long descriptions. Now the full command is written to a temp script (`/tmp/dev3-{taskId}-run.sh`) and tmux receives just `bash "/tmp/..."` — always under the limit.
