Keep tmux shell alive when agent command exits with non-zero code. Previously, if the agent binary was not found or crashed on startup, the tmux session would immediately close, causing a misleading "Worktree not found" error dialog. Now `buildEchoAndRun` and `buildCmdScript` capture the exit code and fall back to an interactive bash shell on failure, matching the existing pattern used by dev server and git operation panes. Additionally, the TaskTerminal error screen now distinguishes between a dead tmux session (worktree still intact) and a truly missing worktree — showing a "Restart Terminal" button in the former case.
