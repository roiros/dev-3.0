Fix dev3 CLI binary not found on app startup for new users. The root cause was using import.meta.dir to locate the bundled CLI binary — in production Electrobun builds, the Bun process runs from Contents/MacOS/ while bundled assets live in Contents/Resources/app/, so the relative path was wrong. Switched to the official Electrobun PATHS.VIEWS_FOLDER API which always resolves correctly. Also fixed the same bug for changelog.json resolution and moved all CLI/skill/PATH setup to run before the potentially slow resolveShellEnv().
