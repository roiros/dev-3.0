Implement the AI Review stage in the task lifecycle. When an AI agent finishes working on a task, a review agent is automatically spawned in a tmux split-pane (30% width, right side) to audit the work — checking for bugs, code reuse issues, and security problems. The reviewer fixes medium/high severity issues and moves the task to User Review. AI Review is configurable per-project (agent, model, prompt) and can be disabled entirely in Project Settings. Agent hooks now support `workingTarget` option so the review agent stays in `review-by-ai` status while working (uses `--if-status` guard), and `skipSystemPrompt` to omit Kanban management instructions from review agents.
