Replaced the three-option terminal keymap selector (Convenient / iTerm2 / Native tmux) with a single iTerm2 compatibility toggle. The two "no shortcuts" options were functionally identical, and tmux native Ctrl+B shortcuts always work regardless of the setting — the only real choice is whether to also enable ⌘D/⌘W/⌘T/⌘[/] shortcuts. Legacy saved values ("dev3", "tmux-native") are normalized to "default" automatically. Also fixed a pre-existing bug where terminalKeymap was never read back from the backend settings file (loadSettings omitted the field), making it localStorage-only; the setting now persists correctly across app restarts.
