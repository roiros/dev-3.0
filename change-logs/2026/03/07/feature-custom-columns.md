Added custom columns to the Kanban board. Each project can define project-scoped columns with a name, color, and LLM instruction (a short hint explaining when an agent should move a task there). Custom columns can be freely positioned anywhere in the board (before, between, or after built-in columns) via drag-and-drop using the handle in the column header; position is stored in `Project.columnOrder`. The `dev3 task move --status <column-id>` CLI command supports custom column IDs, and `dev3 current` shows available custom columns with their instructions. Column management UI lives in Project Settings. Moving a task from `completed` or `cancelled` into a custom column triggers the same resume logic as reopening to an active built-in status (new worktree + PTY, status set to `in-progress`).

Future idea: add `isEndState: boolean` to `CustomColumn` so that end-state custom columns (e.g. "Blocked", "Won't fix") skip the resume/worktree logic entirely when tasks are moved into them. Deferred as over-engineering for now.
